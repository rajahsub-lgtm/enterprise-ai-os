# Enterprise Reasoning Catalog

## Purpose

The Enterprise Reasoning Catalog defines the reasoning methods available to Enterprise AI Operating System (EAIOS).

Reasoning methods represent established enterprise thinking practices that can be selected based on the business problem being solved.

EAIOS orchestrates these methods; it does not replace them.

---

## Problem Analysis

| Method | Description |
|---------|-------------|
| Kepner-Tregoe | Structured root cause analysis |
| Five Whys | Progressive cause identification |
| Fishbone (Ishikawa) | Cause categorization |

---

## Decision Analysis

| Method | Description |
|---------|-------------|
| Kepner-Tregoe | Evaluate alternatives against objectives |
| Weighted Decision Matrix | Compare alternatives objectively |

---

## Risk Analysis

| Method | Description |
|---------|-------------|
| Potential Problem Analysis | Anticipate implementation risks |
| FMEA | Failure mode assessment |

---

## Continuous Improvement

| Method | Description |
|---------|-------------|
| PDCA | Plan, Do, Check, Act |
| Kaizen | Continuous incremental improvement |

---

## Architectural Principles

- Reasoning methods are enterprise assets.
- Reasoning methods are independent of implementation technology.
- AI may augment reasoning methods but does not replace them.
- The catalog will evolve as organizations introduce new enterprise reasoning practices.
